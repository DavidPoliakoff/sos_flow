358  /*
359  * Function..: SOS_pack
360  * Purpose...: Helper function that packs name/val arrays into a handle.
361  * Notes.....: It manages all malloc/free internally, and grows pubs to support
362  * arbitrarily large schemas as needed. This SOS_pack() function
363  * is how you define your schema to SOS.
364  *
365  * The recommended behavior is to pack empty values into each
366  * unique 'name' you intend to publish, and then announce, all
367  * during your program initialization. After that, you can
368  * make calls to SOS_pack() OR SOS_repack() to update values, and
369  * make calls to SOS_publish() to send all changed values out.
370  *
371  * -- PERFORMANCE ADVISORY --
372  *
373  * This function returns the index of the name in the pub's
374  * data store. If you capture that value, you can use it when
375  * making calls to SOS_repack(). This can be considerably faster
376  * depending on your schema's size and ordering. SOS_repack has
377  * O(1) performance with low overhead, vs. O(n) for SOS_pack
378  * and string comparisons across a (potentially) linear scan of
379  * every previously packed data element for a matching name.
380  */
